## ADDED Requirements

### Requirement: Property-based tests for exit code invariants
Using `fast-check` (already in devDependencies), the test suite SHALL include a property-based test in `tests/unit/properties/exit-codes.property.test.ts` that asserts: for any valid command invocation, the resolved exit code is an integer in the range `[0, 10]` (matching the `EXIT_CODES` contract).

#### Scenario: Exit code is always in valid range
- **WHEN** any command action is called with arbitrary valid inputs (generated by fast-check)
- **THEN** the returned exit code SHALL satisfy `Number.isInteger(code) && code >= 0 && code <= 10`

#### Scenario: Exit code property test runs in CI
- **WHEN** `vitest` runs the test suite
- **THEN** the property test in `tests/unit/properties/exit-codes.property.test.ts` SHALL be included and SHALL pass

### Requirement: Property-based tests for FingerprintSchema round-trips
The test suite SHALL include a property-based test in `tests/unit/properties/fingerprint-schema.property.test.ts` that asserts: any `Fingerprint` object generated by a fast-check arbitrary that satisfies `FingerprintSchema` SHALL round-trip through `JSON.stringify` → `JSON.parse` → `FingerprintSchema.parse` without losing data or throwing.

#### Scenario: Fingerprint round-trips through JSON serialization
- **WHEN** a valid `Fingerprint` is serialized to JSON and parsed back
- **THEN** `FingerprintSchema.parse(JSON.parse(JSON.stringify(fingerprint)))` SHALL return an object deeply equal to the original

#### Scenario: Invalid fingerprint fails schema parse predictably
- **WHEN** fast-check generates a fingerprint with a field violating the hardened schema guards (e.g., string length > 512)
- **THEN** `FingerprintSchema.parse()` SHALL throw a `ZodError` (not an unhandled exception of another type)

### Requirement: fast-check arbitraries defined for core domain types
The file `tests/unit/properties/arbitraries.ts` SHALL export fast-check `Arbitrary<T>` instances for `Fingerprint` and `BlueprintIR`. These SHALL be used by all property-based tests to ensure consistent, schema-aware generation.

#### Scenario: Arbitrary generates valid Fingerprint instances
- **WHEN** `fc.sample(fingerprintArbitrary(), 100)` is called
- **THEN** all 100 samples SHALL pass `FingerprintSchema.safeParse()` without errors
